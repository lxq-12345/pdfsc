## 三、配置驱动的工作流程

```
【配置层】协议文件定义规则
  会话管理协议.md → 快照触发：    "快照" / "保存快照"
                   结束归档：    "归档" / "结束会话"

【执行层】脚本读取规则并执行
  snapshot_trigger.sh:
    1. grep "快照触发：" 会话管理协议.md
    2. 提取关键词："快照", "保存快照"
    3. 用户说"快照" → 正则匹配
    4. 生成快照文件

【数据层】快照文件存储
  mem/snapshot/快照-2026-03-11-HHMM.md
```

## 五、关键亮点

✅ **配置与代码分离**
- 快照规则完全由 `会话管理协议.md` 定义
- 脚本只负责读取和执行，无硬编码
- 修改协议即可改变快照行为

✅ **多种实现方式**
- Python 实现（snapshot_handler.py）
- Bash 实现（snapshot_trigger.sh）
- 都能读取同一套配置规则
