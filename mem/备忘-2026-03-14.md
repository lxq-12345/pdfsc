## 五、技术要点总结

### 1. API格式差异处理

**Anthropic格式**（Claude）：
```json
{
  "type": "image",
  "source": {
    "type": "base64",
    "media_type": "image/png",
    "data": "<base64>"
  }
}
```

**OpenAI格式**（Qwen/Ollama）：
```json
{
  "type": "image_url",
  "image_url": {
    "url": "data:image/png;base64,<base64>"
  }
}
```

### 2. Mock测试方法论

**优势**：
- ✅ 无需API Key
- ✅ 零成本
- ✅ 可重复
- ✅ 快速（秒级）

**局限**：
- ⚠️ 无法验证真实API兼容性（已知且可接受）

**缓解措施**：
- 代码已正确实现格式
- Mock测试验证逻辑正确性
- 真实API可用时补充集成测试

### 3. 设计模式应用

1. **工厂模式**（ModelAdapterFactory）
   - 统一创建入口
   - 自动应用能力守护

2. **模板方法模式**（BaseModelAdapter）
   - 统一接口定义
   - 子类实现具体细节

3. **策略模式**（不同适配器）
   - 运行时切换模型
   - 行为封装在适配器内
