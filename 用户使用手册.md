# PDF智转器 用户使用手册

## 0. 手册目标

本手册用于两件事：

1. 指导你从零完成安装、配置、转换、验收。
2. 指导你快速识别“是使用问题、环境问题，还是程序 bug”。

建议优先在 WSL 终端执行命令（Windows 用户）。

## 1. 快速开始（最短路径）

在项目根目录执行：

```bash
bash scripts/install.sh
cp config/config.example.yml config/config.yml
python3 src/pdfsc.py convert ./pdfs/sample.pdf --config ./config/config.yml
python3 src/pdfsc.py verify ./output
```

如果以上四步都通过，说明软件主流程可用。

## 2. 安装与环境检查

### 2.0 从安装包安装（Ubuntu 服务器）

如果你拿到的是发布包（例如 `pdfsc-ubuntu-installer-20260405.tar.gz`），按以下步骤安装：

```bash
# 1) 解压
tar -xzf pdfsc-ubuntu-installer-20260405.tar.gz
cd pdfsc-ubuntu-installer-20260405

# 2) 安装依赖
bash scripts/install.sh

# 3) 初始化配置
cp config/config.example.yml config/config.yml

# 4) 流程自检（推荐先跑）
bash scripts/run_d2.sh flow-test
```

预期结果：

- `flow-test` 通过，说明安装包可在当前 Ubuntu 环境运行。
- 之后可执行 `convert / convert-batch / verify / stats` 进入正式测试。

### 2.1 安装依赖

```bash
bash scripts/install.sh
```

预期结果：

- 输出 `Installation complete!`
- 项目根目录出现 `.venv_wsl/`（WSL）或 `.venv/`（非 WSL）

### 2.2 基础环境自检

```bash
python3 --version
python3 src/pdfsc.py --help
```

预期结果：

- `python3` 可执行
- `pdfsc.py --help` 能显示 `convert / convert-batch / verify / stats`

## 3. 配置说明

### 3.1 创建用户配置

```bash
cp config/config.example.yml config/config.yml
```

### 3.2 三种推荐运行模式

1. 流程联调（推荐先跑通）：`api.provider=mock`
2. 本地模型：`api.provider=ollama`，`api.base_url=http://localhost:11434/v1`
3. 强制离线：命令加 `--offline`

### 3.3 关键参数（最常用）

- `conversion.mode`: `full | restore-only | enhance-only`
- `images.mode`: `disabled | description-only | extract`
- `quality.mark_low_quality_outputs`: 建议保持 `true`
- `paths.output_md`: 输出根目录（默认 `./output/`）

## 4. 使用步骤（Step by Step）

### Step 1：单文件转换

```bash
python3 src/pdfsc.py convert ./pdfs/sample.pdf --config ./config/config.yml --verbose
```

成功判据：

- 终端出现“转换完成”
- 产出文件在 `output/markdown/raw/` 和（full 模式时）`output/markdown/final/`

### Step 2：批量转换

```bash
python3 src/pdfsc.py convert-batch ./pdfs --config ./config/config.yml
```

成功判据：

- 终端出现“批量转换完成”
- 生成 `output/reports/batch_summary_YYYY-MM-DD.md`

### Step 3：质量验证与统计

```bash
python3 src/pdfsc.py verify ./output
python3 src/pdfsc.py stats ./output
```

成功判据：

- `verify` 能输出每个文件评分与是否通过
- `stats` 能输出总文件数、raw/final 数量、总大小

## 5. D2 诊断与修复模式（低成本跑通）

```bash
bash scripts/run_d2.sh flow-test
bash scripts/run_d2.sh convert ./pdfs/sample.pdf mock
bash scripts/run_d2.sh batch ./pdfs mock
```

说明：

- `mock/offline/fallback` 结果允许低质量，仅用于流程验证。
- 这类结果会打标：`DRAFT/LOW_QUALITY`。

## 6. 输出目录解读

- `output/markdown/raw/`：忠实还原结果
- `output/markdown/final/`：增强结果（知识库优先使用）
- `output/reports/`：质量报告、批量汇总
- `output/images/`：仅 `images.mode=extract` 时生成

## 7. Bug 识别指南（重点）

### 7.1 先判断是否“非 bug”

以下通常不是程序 bug：

1. 使用 `mock/offline` 运行，出现低质量文本。
2. 配置了不支持视觉的模型，`images.mode` 被自动降级为 `disabled`。
3. 输入 PDF 本身损坏、加密或无读取权限。

### 7.2 可疑 bug 的典型信号

1. 同一输入、同一配置，多次运行结果结构差异极大。
2. 日志显示“转换完成”，但 `output/markdown/` 没有产物。
3. `verify` 全部失败且失败项集中于同一类结构（如表格全部错位）。
4. `convert-batch` 成功计数与实际产物数量长期不一致。
5. 未使用 `mock/offline`，但结果持续被打 `DRAFT/LOW_QUALITY`。

### 7.3 三步定位法

```bash
# 1) 环境与命令可用性
python3 src/pdfsc.py --help

# 2) 单文件最小复现（加详细日志）
python3 src/pdfsc.py convert <一个最小PDF> --config ./config/config.yml --verbose

# 3) 结构与质量检查
python3 src/pdfsc.py verify ./output
python3 src/pdfsc.py stats ./output
```

判定规则：

- 仅某个 PDF 失败，多为输入样本问题。
- 所有 PDF 在同一阶段失败，多为程序缺陷或配置缺陷。
- 切换 `mock` 正常、`ollama` 失败，多为模型服务/网络链路问题。

## 8. 常见故障与处理

1. `ollama --version` 或 `ollama list` 不可用
- 检查 `ollama` 是否在 PATH
- 检查服务是否已启动：`ollama serve`

2. WSL 中虚拟环境不可用
- 使用 `.venv_wsl`，不要复用 Windows 的 `.venv`

3. 提示“输入文件不存在”
- 检查路径是否相对当前工作目录
- 优先使用绝对路径验证

4. 结果质量明显偏低
- 先确认是否处于 `mock/offline` 模式
- 再确认模型配置是否为预期（provider/model/base_url）
