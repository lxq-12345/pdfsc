| 两步严格分离 | raw版不含增强内容 | ✅ restore仅拼接原文，enhance基于restore二次处理 | ✅ |
| 中间产物保存 | output/markdown/raw/*_raw.md | ✅ `convert_single_pdf()` 中按配置保存 | ✅ |
| 模型调用封装 | 统一`_call_model()`入口 | ✅ 含离线检测、异常捕获、自动回退 | ✅ |
| 本地回退模式 | API不可用时降级运行 | ✅ `_fallback_restore()` + `_fallback_enhance()` | ✅ |
| Frontmatter保证 | 输出必含Frontmatter和一级标题 | ✅ `_ensure_frontmatter_and_title()` 兜底 | ✅ |

**核心架构**：

```
convert_full(pdf_content, metadata)
  ├── convert_restore(pdf_content, metadata)
  │     ├── _build_restore_prompt()     → 构造忠实还原Prompt
  │     ├── _call_model(prompt)         → 调用模型（离线短路 / 异常回退）
  │     └── _fallback_restore()         → 本地回退：Frontmatter + 原文拼接
  └── convert_enhance(restore_markdown, metadata)
        ├── _build_enhance_prompt()     → 构造智能增强Prompt
        ├── _call_model(prompt)         → 调用模型（离线短路 / 异常回退）
        └── _fallback_enhance()         → 本地回退：格式层清理
```

**代码质量**：
- 职责单一，每个方法功能明确
- 调用链清晰：`convert_full` → `convert_restore` + `convert_enhance` → `_call_model` → adapter / fallback
- 防御式编程：`_extract_text()` 兼容 str / dict / pages 多种输入格式
- 离线/在线模式切换透明，上层无感知

**验收结论**：✅ **通过**
