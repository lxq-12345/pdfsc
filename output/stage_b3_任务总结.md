### 3.3 关键发现

#### 3.3.1 API格式差异处理

**发现**：Anthropic和OpenAI的图片输入格式完全不同

**对比表**：
| 特性 | Anthropic | OpenAI Compatible |
|------|-----------|-------------------|
| 图片类型 | `type: image` | `type: image_url` |
| 图片数据位置 | `source.data` | `image_url.url` |
| Base64前缀 | 仅data部分 | 完整 data URI（`data:image/png;base64,...`） |
| media_type位置 | `source.media_type` | URL scheme中 |
| 文本提取路径 | `response.content[].text` | `response.choices[0].message.content` |

**解决方案**：
- 在各适配器的`_build_content()`方法中分别实现
- 代码正确处理了两种格式差异

#### 3.3.2 Mock测试方法论

**优势**：
- 无需API Key即可验证格式正确性
- 快速反馈（秒级完成全部测试）

- 测试脚本成功运行

**状态**：✅ 已解决

---

### 4.2 问题2：capabilities参数类型错误

**现象**：传入dict导致 `'dict' object has no attribute 'get_vision_support'`

**错误代码**：
```python
adapter = AnthropicAdapter(
    ...
    capabilities={'vision': True}  # ❌ 错误：传入dict
)
```

**根因**：`check_vision_support()`期望`ModelCapabilities`对象，而非dict

**解决方案**：
```python
capabilities = ModelCapabilities()
capabilities.load_registry('config/model_capabilities.yml')
adapter = AnthropicAdapter(
    ...
    capabilities=capabilities  # ✅ 正确：传入对象
)
```

**状态**：✅ 已解决

---

### 4.3 问题3：Mock测试call_count累积

**现象**：多次测试导致`mock_client.messages.create.call_count`不准确

**根因**：多个测试用例共用同一个mock对象

**解决方案**：
```python
# ❌ 错误：共用mock对象
mock_client = Mock()
test1()  # call_count = 1
test2()  # call_count累积

# ✅ 正确：每个测试用独立mock对象
def test1():
    mock_client = Mock()  # 独立对象
    ...

def test2():
    mock_client = Mock()  # 独立对象
    ...
```

**状态**：✅ 已解决
