## 3. 执行前准备

### 3.1 远程与系统准备

1. 使用 ToDesk 登录 Ubuntu 服务器。
2. 确认当前用户具备 sudo 权限。
3. 记录服务器基础信息：

建议命令：

```bash

hostnamectl
lsb_release -a || cat /etc/os-release
nproc && free -h && df -h
```

### 3.2 目录准备

建议统一目录：

```bash
mkdir -p ~/projects
cd ~/projects
```

### 3.3 基准 PDF 文件准备（必须）

在开始 D2 前，必须先准备 3 个基准 PDF，统一放置到服务器目录：

`~/projects/pdfsc-ubuntu-installer-20260405/pdfs_d2_baseline/`

#### 3.3.1 基准文件清单

1. `01_华为TaiShan_产品概述.pdf`
2. `02_TaiShan200_快速安装指南_型号2280.pdf`
3. `03_TaiShan200_用户指南_型号2280.pdf`

说明：若原始文件名不同，上传后请重命名为以上标准名，便于验收对照与批量统计。

#### 3.3.2 上传后校验（防错传/漏传）

```bash
cd ~/projects/pdfsc-ubuntu-installer-20260405
mkdir -p pdfs_d2_baseline
ls -lh pdfs_d2_baseline
find pdfs_d2_baseline -maxdepth 1 -type f -name "*.pdf" | wc -l
sha256sum pdfs_d2_baseline/*.pdf
```

通过标准：

1. PDF 数量必须为 3。
2. 文件非 0 字节且可正常读取。
3. SHA256 已记录，便于复现一致性。

## 8. 缺陷分级与处理策略

### 8.1 缺陷分级

1. **P0**：无法运行/崩溃/无输出
2. **P1**：结果严重错误（结构错乱、大片缺失）
3. **P2**：质量退化（可用但不理想）
4. **P3**：文档/体验问题

### 8.2 处理规则

1. 出现 P0/P1：立即停在当前轮，先修复再继续。
2. P2：记录问题并可继续执行后续样本。
3. P3：归档到优化清单，不阻塞验收主线。
