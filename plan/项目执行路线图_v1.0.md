## 二、各阶段详细定义

### 准备阶段（Preparation）

#### Step1：项目目录结构创建 ✅
- **状态**：已完成

- **目的**：为前置验证阶段（Step0-1/0-4）准备高质量测试样本
- **输入**：

  ├── level2_moderate_expected.md

### 前置验证阶段（Pre-Dev Validation）

#### Step0-1：瓶颈1验证 - 表格语义转换 【**必须通过**】

**验证目标**：
- 确认表格语义转换提示词效果稳定
- 三个层次的转换质量均达标
- 接近已完成文件的转换质量

**测试案例**：使用Step3制作的三层次样本

**验证方法**：
- 使用Claude手动转换（未来用阶段A工具自动转换）
- 对比预期输出，按《Markdown转换规范.md》审查清单评分
- 连续3次转换均≥9.7分 = 通过

**通过标准**：
```
单次转换评分：≥9.7分（优秀）
  ├─ 审查清单14项全部✅
  ├─ 表格无传统MD表格格式
  └─ 内容100%对应原文

连续验证：第1、2、3次均≥9.7分 = 瓶颈1验证通过
```

**评分细则**：见 `mem/瓶颈1评分标准_2026-03-11.md`

**预计工作量**：1天（含样本制作、手动转换、评分）

---

#### Step0-4：瓶颈4验证 - 防幻觉约束 【**配合Step0-1**】

**验证目标**：
- 确认防幻觉四层保障体系有效
- 忠实还原版与原文100%对应
- 增强版无等级3幻觉内容

**测试案例**：使用Step3制作的风险样本（容易幻觉的内容片段）

**验证条件**：
- 须在阶段B（AI核心）完成后进行

**关键问题**：【待用户确认】
