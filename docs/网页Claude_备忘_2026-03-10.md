### 五个顶级目录的角色
```
config/   →  工具行为配置（程序怎么运行）
skills/   →  AI技能规范（AI遵循什么标准）
prompts/  →  AI执行提示词（AI怎么执行任务）
plan/     →  项目规划和决策文档（给人看）
mem/      →  备忘和开发日志
output/   →  转换输出和验收报告
tests/    →  测试素材和基准
```

### skills/目录的定位
```
Markdown转换规范.md → 定位为AI的"宪法"
  不是提示词
  不是配置文件
  是所有提示词的依据和标准
  稳定，很少改动

与prompts/的关系：
  规范（宪法）→ 被提示词引用
  提示词 → 经常调整迭代
  两者独立演进，互不干扰
```

### 运行时加载顺序
```
skills/markdown_conversion.md（规范）
  +
prompts/restore.md 或 enhance.md（提示词）
  +
prompts/examples/table_level{N}.md（示例）
  +
当前PDF内容
  ↓
动态组装，发送给AI
```

### CLAUDE.md vs README.md
```
CLAUDE.md  →  给Claude Code看，启动时自动读取
               内容：决策约束、执行边界、当前任务
               现在就建

README.md  →  给人类开发者看，项目完成后生成
               内容：安装方法、使用说明
               阶段E完成后再建，可从文档自动生成
```

### 配置文件体系（三层次）
```
层次1：config.yml（完整配置）
  → 所有配置项，用户直接编辑
  → 适合首次设置和复杂配置

层次2：菜单快捷切换（常用配置）
  → 模型、图片模式、输出目录
  → 自动保存到config.yml

层次3：转换时临时参数（一次性）
  → 只影响本次转换，不修改配置文件
```

### 集成方式
```
大模型调用   →  API调用（HTTP/REST）
Python工具库 →  直接import调用
外部服务扩展 →  MCP协议（未来扩展方向）
```
