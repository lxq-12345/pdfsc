# 会话备份与快照机制 - 开发者指南

**版本**：1.0 | **日期**：2026-03-11 | **面向对象**：Claude项目开发使用者

---

## 一、概述

本项目实现了**自动会话备份机制**，确保任何会话中断时：
- ✅ 最多丢失15分钟内容
- ✅ 会话结束时自动生成结构化备忘
- ✅ 支持多Claude窗口并发（完全隔离）

---

## 二、核心概念

### 2.1 会话快照（Snapshot）

**定义**：某时间段内的原始对话记录

```
目的：防止意外中断导致内容丢失
格式：原始对话，不加工
周期：用户自主触发（关键词"快照"）
命名：快照-YYYY-M-D-HHMM-[SESSION_ID].md
位置：mem/snapshot/
```

**示例**：
```
快照-2026-03-11-1232-20260311_1232_a7f3.md

内容：
时间段：12:00 - 12:32

用户：修改会话管理协议.md的问题
Claude：理解反馈，需要精简为操作指令风格...

用户：快照
Claude：✅ 快照已生成
```

### 2.2 会话备忘（Memo）

**定义**：一整天的对话摘要，会话结束时生成

```
目的：供下次会话快速恢复上下文
格式：结构化（主题、决策、文件变更、待办、下一步）
周期：当天说"归档"时生成
命名：备忘-YYYY-M-D.md
位置：mem/
保留：永久保留（历史记录）
```

**示例**：
```
备忘-2026-03-11.md

# 会话备忘
日期：2026-03-11
时间段：11:35 - 12:45
参与方：用户 + Claude Code（本地）
```

### 2.3 会话ID（SESSION_ID）

**定义**：每次启动自动生成的唯一会话标识

```
格式：YYYYMMDD_HHMM_XXXX
例：20260311_1232_a7f3
  20260311  → 日期
  1232      → 启动时间（时分）
  a7f3      → 随机4位16进制数

用途：多窗口并发时隔离缓冲和快照文件，防止冲突
```

## 三、工作流程

### 3.1 启动流程

```
【新会话启动】
   ↓
【自动执行1.1-1.4】
   ├─ 生成SESSION_ID（如：20260311_1232_a7f3）
   ├─ 检查前日未归档快照 → 自动生成备忆
   ├─ 初始化缓冲文件 mem/.snapshot_buffer_[SESSION_ID].md
   ├─ 缓存触发关键词
   └─ 首次快照保存前一会话内容
   ↓
【系统就绪】
```

### 3.2 对话流程

```
【用户输入信息】
   ↓
【自动追加到缓冲】
缓冲文件：mem/.snapshot_buffer_[SESSION_ID].md
追加格式：
  用户：消息内容
  Claude：回应内容
   ↓
【继续下一条消息】
```

### 3.3 快照触发

```
【用户说："快照"】
   ↓
【自动识别关键词】
   ↓
【快照生成操作】
   ├─ 读取缓冲内容
   ├─ 生成快照文件：mem/snapshot/快照-YYYY-M-D-HHMM-[SESSION_ID].md
   ├─ 清空缓冲（防止重复）
   └─ 初始化新周期
   ↓
【通知用户】"✅ 快照已生成：快照-2026-03-11-1300-20260311_1232_a7f3.md"
```

### 3.4 结束流程

```
【用户说："归档"】
   ↓
【备忘生成操作】
   ├─ 扫描mem/snapshot/ → 找出当日所有快照
   │  （包括所有SESSION_ID的快照）
   ├─ 按时间顺序合并对话内容
   ├─ 提取关键信息：
   │  - 讨论主题
   │  - 关键决策
   │  - 文件变更
   │  - 待办事项
   │  - 下一步方向
   └─ 生成备忆文件：mem/备忘-YYYY-M-D.md
   ↓
【通知用户】"✅ 备忘已生成：备忆-2026-03-11.md"
```

## 五、常见操作

### 5.1 手动保存快照

```
【场景】做完一个阶段的工作，想保存进度

【操作】
用户：快照

【系统反应】
✅ 快照已生成：mem/snapshot/快照-2026-03-11-1232-20260311_1232_a7f3.md
```

### 5.2 结束会话并归档

```
【场景】今天工作结束，生成备忘供明天快速恢复

【操作】
用户：归档

【系统反应】
✅ 备忘已生成：mem/备忘-2026-03-11.md
（会话结束）
```

### 5.3 中断后恢复上下文

```
【场景】会话意外中断（网络断开、进程崩溃等）

【启动新会话后操作】
用户：继续[项目名]

【系统反应】
1. 自动加载最新快照（当前SESSION_ID）
2. 恢复上次中断的对话历史
3. 通知："✅ 已加载最新快照，从此处继续"
```

### 5.4 查看备忘恢复上下文

```
【场景】隔天开始工作，想快速回顾前日工作内容

【操作】
1. 打开 mem/备忘-2026-03-10.md
2. 查看"五、下一步"部分 → 知道今天要做什么
3. 相关快照在"六、参考快照"中列出
```

### 5.5 多窗口并发工作

```
【场景】同时打开两个Claude窗口，处理不同的任务

【工作流程】
窗口A：项目PDF转换              窗口B：文档编写
启动 ↓                            启动 ↓
SESSION_ID_A                      SESSION_ID_B
缓冲文件A（隔离）                缓冲文件B（隔离）
快照A（隔离）                    快照B（隔离）
   ↓                               ↓
完全独立        ✅               完全独立
无任何冲突      ✅               无任何冲突
可同时运行      ✅               可同时运行

【结束时】
两个窗口都说"归档" → 当日备忘同时包含A和B的全部内容
```

## 八、故障排查

### 问题1：快照文件没有生成

**症状**：说了"快照"但没有生成文件

**检查清单**：
1. 确认说的关键词正确
   - ✓ "快照" 或 "保存快照"
   - ✗ "做快照"、"保存一下" 等变体

2. 检查文件权限
   ```bash
   ls -la mem/snapshot/
   # 应该有读写权限 (drwxr-xr-x)
   ```

3. 检查缓冲文件有内容
   ```bash
   cat mem/.snapshot_buffer_*.md
   # 应该有用户和Claude的对话记录
   ```

4. 查看最新快照文件
   ```bash
   ls -lt mem/snapshot/ | head -5
   # 最新的快照文件应该有内容
   ```

### 问题2：快照内容为空或不完整

**症状**：快照文件存在但内容不完整

**原因**：可能缓冲在快照前没有被清理

**解决**：
1. 检查缓冲文件是否被清空
2. 手动执行再次快照
3. 查看两个快照文件，内容应该不重复

### 问题3：多窗口快照混乱

**症状**：快照中混合了不同窗口的内容

**原因**：未使用SESSION_ID隔离（v2.0遗留问题）

**解决**：确保运行v2.1版本
```bash
grep "version:" 会话管理协议.md
# 应该显示 version: 2.1
```

### 问题4：备忘文件没有生成

**症状**：说了"归档"但备忘没有生成

**检查清单**：
1. 确认关键词正确："归档" 或 "结束会话"
2. 检查当日是否有快照文件
   ```bash
   ls mem/snapshot/ | grep 2026-03-11
   ```
3. 确保mem/目录有写权限
4. 手动查看最新一个快照是否有完整内容

## 十三、最佳实践

### DO  ✅

- ✅ **定期保存快照**（每15-30分钟）
  - 防止突发中断丢失内容

- ✅ **工作阶段结束时快照**

### DON'T ❌

  - 会破坏完整性

  - 应分开到不同窗口

- ❌ **不要忘记运行"继续"来恢复**
  - 手动重复工作会浪费时间

## 十四、常见问题 FAQ

**Q：快照和备忘的区别是什么？**
A：快照是**原始记录**（防止丢失），备忘是**结构化摘要**（便于回顾）。

**Q：可以有多个备忘文件吗？**
A：一天一份。同一天多次会话时，追加到同一个备忘文件。

**Q：SESSION_ID是必需的吗？**
A：必需。用于多窗口隔离。即使只有一个窗口，也会自动生成。

**Q：快照文件可以删除吗？**
A：原则上可以，但建议保留至少一周（备忘需要引用）。

**Q：如何导出快照和备忘？**
A：直接复制mem/目录下的.md文件即可（文本格式）。

**Q：支持跨天会话吗？**
A：支持。按**启动日期**归档。23:00启动、次日01:00结束的会话，按启动日期(前一天)归档。
