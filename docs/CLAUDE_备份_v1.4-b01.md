### 沟通效率规范（强制执行）

**遇到模糊指令的处理原则**：

**1. 立即反问，不猜测**
   - 识别模糊指令后，2秒内反问
   - 不要试图通过复杂推理"猜测"用户意图
   - 快速互动优于沉默推理

**2. 模糊指令识别信号**
   - 动词不明确："继续"、"准备"、"下一步"
   - 目标对象不清晰
   - 可以有2种以上合理解释
   - 缺少关键上下文信息

**3. 简洁反问格式**
   ```
   "请明确：[选项1] 或 [选项2]？"
   ```
   - 最多3个选项
   - 用最简单的语言
   - 允许用户用自然语言回答

**4. 区分讨论与执行**
   - **讨论类特征**："讨论："前缀、问句形式、征求意见
   - **执行类特征**："执行"、"立即"、"创建"、动词明确
   - **默认保守**：不确定时假设为"讨论"，先回答再确认是否执行

**示例**：
```
用户："继续下一步"
我：  "请明确'下一步'：1. 仅标记Step0-1完成 或 2. 开始Step0-4执行？"

用户："准备XXX"
我：  "是讨论XXX方案，还是立即开始制作XXX文件？"
```

每次会话启动时，优先于一切任务，读取并严格执行：
`./会话管理协议.md`

### 归档提醒责任

**你必须在以下情况主动提醒用户归档**：

1. **阶段性任务完成后** - 完成Step任务或阶段任务后
2. **用户表示要结束会话时** - 检测到"好的"、"可以了"、"今天就这样"等结束意向
3. **生成重要报告后** - 完成任务总结、验收报告等文档

**提醒格式示例**：
```
📌 **提醒**：今日已完成以下任务：
- [任务1]
- [任务2]

是否需要归档备忘？（说"归档"即可生成备忘文件）
```

**注意**：
- 不要等用户主动说"归档"，阶段性成果完成后应主动提醒
- 提醒应自然、简洁，不过于频繁
- 用户明确拒绝时，不重复提醒

### 会话中断恢复

如果会话中断（如网络断开、进程崩溃），启动新会话时：

1. 自动执行 `会话管理协议.md` 中的检查流程
2. 扫描 `mem/snapshot/` 下的快照文件
3. 与当前日期对比，若当天有未归档快照 → 自动恢复上下文
4. 向用户报告"已恢复"或"需手动确认"的部分
