## 4. 路径与软链接配置（WSL）

### 4.1 Ollama 两个目录的职责

- `/home/wy885/ollama-bin`：Ollama 程序本体与运行时库
- `/home/wy885/.ollama`：模型与状态数据（manifests + blobs + 密钥）

说明：
- 两目录互相关联但职责不同，不建议单独删除其一。
- 删 `ollama-bin` 会失去可执行程序；删 `.ollama` 会丢失已下载模型。

### 4.2 已观测到的关键软链接

项目目录下（`/mnt/d/usr/百信公司项目/pdfsc`）：
- `./.venv_wsl/bin/python -> python3.12`
- `./.venv_wsl/bin/python3 -> python3.12`
- `./.venv_wsl/bin/python3.12 -> /usr/bin/python3.12`
- `./.venv_wsl/lib64 -> lib`

Ollama 运行库目录下（`/home/wy885/ollama-bin/lib/ollama`）：
- `include`（符号链接）
- `libggml-base.so`（符号链接）
- `libggml-base.so.0`（符号链接）

### 4.3 PATH 配置位置

## 5. 虚拟环境约定（强烈建议）

### 4.1 约定原则

- Windows Python 使用 Windows venv（如 `.venv`）
- WSL Python 使用 Linux venv（建议 `.venv_wsl`）
- 两者不要混用，不要共用同一个解释器路径

### 4.2 推荐目录约定

- Windows venv：`.venv`
- WSL venv：`.venv_wsl`

## 6. 快速自检命令（WSL）

```bash
# 1) WSL版本与发行版
wsl.exe --status
wsl.exe -l -v

# 2) 项目路径可达
wsl.exe bash -lc "ls -ld '/mnt/d/usr/百信公司项目/pdfsc'"

# 3) WSL Python
wsl.exe bash -lc "python3 --version"

# 4) 检查WSL venv
wsl.exe bash -lc "cd '/mnt/d/usr/百信公司项目/pdfsc' && test -x .venv_wsl/bin/python && .venv_wsl/bin/python --version || echo 'missing:.venv_wsl'"

# 5) 检查容器/模型命令
wsl.exe bash -lc "command -v docker || echo 'docker:missing'; command -v docker-compose || echo 'docker-compose:missing'; command -v ollama || echo 'ollama:missing'"
```

## 7. 常见问题

### 7.1 WSL 下 `.venv` 不可用

现象：
- `.venv` 存在，但执行 `./.venv/bin/python` 报不存在

原因：
- `.venv` 是 Windows 格式（`Scripts/`），不是 Linux 格式（`bin/`）

处理建议：
- 在 WSL 内单独创建并使用 `.venv_wsl`

### 7.2 `wsl.exe bash -lc` 报 `E_ACCESSDENIED`

现象：
- 在受限终端中调用 WSL 命令出现权限拒绝

处理建议：
- 在具备权限的终端重试，或提升当前终端执行权限后再检查

### 7.3 `ollama` 能识别但 `ollama list` 超时

现象：
- `command -v ollama` 有输出
- `ollama list` 报 `127.0.0.1:11434` 超时

原因：
- 客户端已就绪，但 `ollama serve` 服务未启动

处理建议：
- 先启动服务，再执行 `ollama list`
