```
BaseModelAdapter (基类)
```

### 3.2 核心接口设计

**BaseModelAdapter 统一接口**：

| 方法 | 功能 | 返回值 |
|------|------|--------|
| `call(prompt, images, max_retries)` | 调用模型（抽象方法） | str |
| `check_vision_support()` | 检查模型是否支持视觉 | bool |
| `apply_capability_guard()` | 能力守护（自动降级） | bool |
| `_prepare_images(images)` | 预处理图片输入 | list |
| `_encode_image(image)` | Base64编码图片 | tuple |
| `_resolve_api_key()` | 解析API Key | str |

### 3.3 适配器实现对比

#### AnthropicAdapter（Claude API）

**特点**：
- 请求格式：Anthropic Messages API
- 图片格式：`type: image` + `source.data`（纯Base64）
- 文本提取：从 `response.content[].text` 提取
- 默认环境变量：`ANTHROPIC_API_KEY`

**代码位置**：235-308行

#### QwenAdapter（阿里通义千问）

**特点**：
- 继承自：OpenAICompatibleAdapter
- 请求格式：OpenAI Chat Completions API
- 图片格式：`type: image_url` + `image_url.url`（完整data URI）
- 默认Base URL：`https://dashscope.aliyuncs.com/compatible-mode/v1`
- 默认环境变量：`DASHSCOPE_API_KEY`

**代码位置**：401-405行

#### OllamaAdapter（本地模型）

**特点**：
- 继承自：OpenAICompatibleAdapter
- 请求格式：OpenAI Chat Completions API
- 默认Base URL：`http://localhost:11434/v1`
- 无需API Key（本地调用）

**代码位置**：408-411行

### 3.4 错误处理机制

**重试逻辑**（所有适配器统一）：
- 默认重试次数：3次（可配置）
- 重试间隔：递增延迟（1s、2s、4s...）
- 最终失败：抛出 `RuntimeError` 并包含详细错误信息

**代码实现**（model_adapter.py 278-308行）：
```python
for attempt in range(retries):
    try:
        response = client.messages.create(...)
        return self._extract_text(response)
    except Exception as exc:
        last_error = exc
        if attempt == retries - 1:
            break
        time.sleep(self._retry_delay(attempt))

raise RuntimeError(f'调用失败: {last_error}') from last_error
```

### 3.5 工厂模式实现

**ModelAdapterFactory**（184-232行）：

- `create_adapter()` - 指定provider和model创建适配器
- `create_from_config()` - 从配置对象直接创建适配器
- `ADAPTER_MAP` - 适配器注册表

**优势**：
- 统一创建入口，简化调用代码
- 自动应用能力守护机制
- 支持从配置对象直接创建

## 七、技术亮点

### 7.1 设计模式应用

1. **工厂模式**（ModelAdapterFactory）
   - 统一创建入口
   - 自动应用能力守护
   - 简化调用代码

2. **模板方法模式**（BaseModelAdapter）
   - 统一接口定义
   - 子类实现具体细节
   - 代码复用程度高

3. **策略模式**（不同适配器）
   - 运行时切换模型
   - 行为封装在适配器内

### 7.2 关键技术点

1. **API格式差异处理**
   - 图片编码：Anthropic vs OpenAI格式
   - 请求结构：Messages API vs Chat Completions API
   - 响应解析：不同字段路径

2. **错误处理与重试**
   - 指数退避重试策略
   - 详细异常链保留
   - 可配置重试次数

3. **能力检测与降级**
   - 自动检测模型视觉能力
   - 不支持时自动禁用图片模式
   - 避免运行时错误

### 7.3 Mock测试方法论

**优势**：
- 零成本验证代码正确性
- 快速反馈（秒级）
- 可重复执行
- 不依赖外部服务

**局限**：
- 无法验证真实API兼容性
- 无法测试网络问题

**缓解措施**：
- 真实API可用时补充集成测试
- 代码已正确实现格式，mock验证逻辑
