## 4. Ollama 路径配置

### 4.1 两个目录的职责

- `/home/wy885/ollama-bin`：Ollama 程序本体与运行库
- `/home/wy885/.ollama`：模型与状态数据（manifests、blobs、密钥）

两者关系：
- `ollama-bin` 是可执行与运行时
- `.ollama` 是模型仓库
- 两者缺一都会影响可用性

### 4.2 PATH 配置（WSL）

配置文件：
- `~/.bashrc`
- `~/.profile`

配置行：

```bash
export PATH=/home/wy885/ollama-bin/bin:$PATH
```

验证命令：

```bash
command -v ollama
printenv PATH
```

## 5. Python 与虚拟环境约定

- `.venv`：Windows 风格虚拟环境（`Lib/`、`Scripts/`）

约定：
- Windows 下使用 `.venv`
- WSL 下使用 `.venv_wsl`
- 不跨环境复用同一个 venv

## 8. 常见问题

### 8.1 WSL 下 `.venv` 不能用

原因：
- `.venv` 为 Windows 结构，不含 Linux 的 `bin/python`

处理：
- 在 WSL 使用 `.venv_wsl`

### 8.2 `ollama` 能识别但 `ollama list` 超时

原因：
- 客户端在 PATH 中，但服务未启动

处理：
- 先启动 `ollama serve`，再执行 `ollama list`

### 8.3 `wsl.exe bash -lc` 报 `E_ACCESSDENIED`

原因：
- 当前终端调用权限受限

处理：
- 在具备权限的终端执行，或提升当前终端权限后重试

如软链接异常，必须暂停并报告，不得继续假设执行。
