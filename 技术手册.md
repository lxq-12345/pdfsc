# PDF智转器 技术手册

## 1. 目标与范围

PDF智转器（`pdfsc`）用于将 PDF 转换为 Markdown，面向知识库/RAG 场景。  
本手册说明系统设计、关键模块、配置策略与运行机制，供开发与维护使用。

## 2. 核心流程

系统采用两步流水线：

1. 忠实还原（restore/raw）
- 输入提取后的 PDF 内容
- 输出与原文对应的结构化 Markdown

2. 智能增强（enhance/final）
- 基于 restore 结果进行表达与结构优化
- 在 `offline/fallback/mock` 场景自动打 `DRAFT/LOW_QUALITY` 标记

## 3. 模型调用策略

支持提供商：
- `anthropic`
- `openai`
- `qwen`
- `ollama`
- `mock`（流程联调用）

策略说明：
- `offline_mode=true`：跳过外部模型调用，进入本地回退
- `mock_mode=true` 或 `provider=mock`：使用本地 mock 输出
- 模型不支持视觉时自动降级 `images.mode=disabled`

## 4. 配置体系

配置优先级：
1. CLI 参数
2. 用户配置（`config.yml`）
3. 默认配置（`config/default.yml`）

关键配置：
- `api.provider / api.model / api.base_url`
- `api.offline_mode / api.mock_mode`
- `conversion.mode`
- `images.mode`
- `quality.mark_low_quality_outputs`

## 5. 输出与报告

默认输出：
- `output/markdown/raw/`
- `output/markdown/final/`
- `output/reports/`

批量汇总报告包含：
- 低质量命中统计
- restore/enhance 模式分布
- 单文件模式与评分摘要

## 6. D2 流程联调

推荐入口：

```bash
bash scripts/run_d2.sh flow-test
```

支持命令：

```bash
bash scripts/run_d2.sh convert <input.pdf> [mock|offline|real]
bash scripts/run_d2.sh batch <input_dir> [mock|offline|real]
```
