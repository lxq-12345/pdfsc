## 一、项目简介

一个 AI 驱动的 PDF → Markdown 智能转换工具。  
中文名：PDF智转器 | 英文ID：pdf-smart-converter | CLI工具名：pdfsc

---

## 二、你的角色

**你是执行者，不是决策者。**

遇到规划未覆盖的情况，暂停并报告，不要自行填充决策。

## 三、必读文档（按顺序）

启动后请按以下顺序阅读，建立完整认知后再执行任务：

**所有文件路径均相对于项目根目录**：

```
1. ./会话管理协议.md
   → 启动检查流程、快照与备忘机制、中断恢复流程
   → 每次会话都需遵循，优先级最高
   → 首次阅读需完整理解所有章节

2. ./plan/项目规划.md
   → 完整技术方案，重点阅读章节（更新于 2026-03-10）：
     第二章 - 项目文件结构
     第五章 - 开发阶段
     第九章 - 技术瓶颈验证任务
     第十章 - 验收标准
     第十三章 - 版本管理规范
     第十四章 - 环境准备与前置条件
   ⚠️ 如果项目规划有重大版本更新，优先阅读"修订历史"

3. ./skills/Markdown转换规范.md
   → Markdown转换规范（AI行为宪法）
   → 所有转换任务的直接执行依据
   → 开发提示词前必须完整读懂

4. ./mem/ 目录下日期最新的备忘文件
   → 最新讨论成果、当前待办、下一步方向
```

---

## 四、当前任务

**优先级顺序**：

1. 如果 `mem/` 目录存在最新备忘文件（按文件日期）
   → 查看文件中"下一步方向"章节

2. 如果 `mem/` 目录为空或无备忘文件
   → 参考 `plan/项目规划.md` 第五章"开发阶段"，从**阶段A**开始

## 五、遇到问题

**以下情况立即暂停，向用户报告而非自行决策**：

- 项目规划未涵盖的技术方案选择题
- 多种实现方案利弊无法依据现有文档判断
- 质量标准有歧义或不同文档间有冲突

**处理步骤**：

1. 优先查阅 `plan/项目记忆文档.md` 寻求已知解决方案
2. 如果仍无答案，向用户清晰陈述问题、上下文、以及可选方案
3. 等待用户指示，勿自行填充决策

**不要自行决策，不要降低质量标准。**

## 八、会话中断恢复

如果会话中断（如网络断开、进程崩溃），启动新会话时：

1. 自动执行 `会话管理协议.md` 中的检查流程
2. 扫描 `mem/snapshot/` 下的快照文件
3. 与当前日期对比，若当天有未归档快照 → 自动恢复上下文
4. 向用户报告"已恢复"或"需手动确认"的部分

## 九、反馈与改进

如果发现本配置文件（CLAUDE.md）或其他文件有以下问题：

- 表述不清楚或存在歧义
- 指导不完整或缺少必要的降级方案
- 不同章节间有逻辑矛盾

**处理方式**：

1. 直接向用户报告问题位置和影响范围
2. 不要自行修改配置文件
3. 用户可以选择即时修改或记录到 `plan/项目记忆文档.md` 作为优化案例
