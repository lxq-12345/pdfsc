# 会话管理协议

**目标**：防止会话中断导致内容丢失；会话结束时生成备忘供下次快速恢复；主动提醒用户归档。

---

## 一、自动启动指令

每次Claude启动新会话时，**立即执行以下流程**（无需用户触发）：

### 1.1 会话隔离初始化
```
【会话ID生成】
  生成本次会话的唯一ID：SESSION_ID = YYYYMMDD_HHMM_XXXX
  例：20260311_1232_a7f3
  用途：隔离多窗口并发的缓冲和快照文件，防止冲突
```

### 1.2 检查阶段
```
① 读取 mem/snapshot/ 目录中的最新快照文件
② 对比快照日期与TODAY：
   - 若日期 < TODAY：【强制执行】自动生成前日备忆文件
     → 读取前日所有快照
     → 整理为结构化备忘
     → 保存至 mem/备忘-[前日日期].md
     → 向用户报告："✅ 已自动生成前日备忘（YYYY-MM-DD）"
   - 若日期 = TODAY：检查是否有未归档快照
     → 若存在多个快照且无备忘，提示："⚠️ 检测到今日已有会话但未归档"
③ 缓存触发关键词（见第二章）
```

### 1.3 启动快照与缓冲机制（自动强制执行）
```
【强制执行】启动时自动执行以下操作：
  ① 初始化缓冲文件：mem/.snapshot_buffer_[SESSION_ID].md
  ② 开始记录本次会话对话到缓冲文件
  ③ 向用户报告："✅ 会话已启动（Session ID: YYYYMMDD_HHMM_XXXX），缓冲机制已激活"
```

### 1.4 就绪状态
```
系统就绪，等待用户任务输入
缓冲机制已运行，自动记录本次会话对话（隔离在本SESSION_ID下）
用户可随时说"快照"保存当前进度
```

## 二、触发词汇表（配置驱动）

识别用户消息中的关键词，自动触发对应操作：

```
「快照触发」
  关键词：     "快照" / "保存快照"
  触发操作：   读取缓冲 → 生成快照文件 → 清空缓冲 → 初始化新周期
  返回提示：   "✅ 快照已生成：mem/snapshot/快照-YYYY-M-D-HHMM-[SESSION_ID].md"

「结束归档」
  关键词：     "归档" / "结束会话"
  触发操作：   整理当日所有快照文件（包括本SESSION_ID的） → 生成结构化备忘 → 标记会话结束
  返回提示：   "✅ 备忘已生成：mem/备忘-YYYY-M-D.md"

「中断恢复」
  关键词：     "继续[项目名]" / "恢复上一个session"
  触发操作：   加载本SESSION_ID的最新快照 → 恢复上次中断的上下文
  返回提示：   "✅ 已加载最新快照，从此处继续"
```

## 三、工作流程

```
会话启动
  ↓
【自动】执行启动检查流程 + 启动快照与缓冲机制（第一章）
  ↓
【自动】报告："✅ 会话已启动，缓冲机制已激活"
  ↓
【可选】任何时刻用户可说"快照"保存进度
  ↓
【可选】会话中断后说"继续"恢复上下文
  ↓
【自动提醒】阶段性任务完成后提醒归档（见3.1节）
  ↓
【手动】会话结束说"归档"生成备忘
```

### 3.1 自动归档提醒机制

**触发时机**：
Claude应在以下情况主动提醒用户归档：

1. **阶段性任务完成时**
   - 完成一个Step任务（如Step0-1, Step0-4）
   - 完成一个阶段（如阶段A1, A2）
   - 生成重要文档或报告后

2. **用户表示要结束会话时**
   - 用户说"好的"、"可以了"、"今天就这样"等结束意向
   - 用户询问"下一步做什么"后没有继续执行

3. **会话较长时的定期提醒**
   - 完成多个独立任务后（建议每3-5个任务提醒一次）
   - 会话超过1小时时（如适用）

**提醒格式**：
```
📌 **提醒**：今日已完成以下任务：
- [任务1]
- [任务2]

是否需要归档备忘？（说"归档"即可生成备忘文件）
```

**注意事项**：
- 提醒应自然融入对话，不应过于频繁
- 优先在阶段性成果完成后提醒
- 用户明确拒绝时，不重复提醒

## 四、中断恢复

```
发生中断后：

【恢复上次会话】
  用户："继续[项目]"
    ↓
  Claude：加载 mem/snapshot/ 中最新的快照文件
    ↓
  自动恢复上次中断的上下文和对话历史
    ↓
  用户："✅ 已恢复，从这里继续"

【重新开始】
  用户："开始新session"
    ↓
  Claude：忽略快照，开始一个全新会话
```

---

## 五、文件位置

```
【会话隔离】
  每次启动自动生成唯一会话ID：SESSION_ID = YYYYMMDD_HHMM_XXXX
  多窗口并发时，各自拥有独立的缓冲和快照命名空间

会话快照：  mem/snapshot/快照-YYYY-M-D-HHMM-[SESSION_ID].md
                     ↑
               带会话ID后缀，同分钟多窗口仍可独立生成

缓冲文件：  mem/.snapshot_buffer_[SESSION_ID].md
                     ↑
               每个会话独立缓冲，互不干扰

会话备忘：  mem/备忘-YYYY-M-D.md
              （保持不变，因为归档时会扫描当日所有快照，
               包括所有SESSION_ID的快照，统一整理）

【文件管理】
- 所有快照和备忘文件均自动保留（不自动删除）
- 用户需手动删除时可指示Claude删除
```
