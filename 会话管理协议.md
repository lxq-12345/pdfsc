# 会话管理协议

## 目标

确保任何会话在意外中断时，最多丢失15分钟的内容。  
会话结束时生成结构化备忘，供下次会话快速恢复上下文。

---

## 一、两个核心概念

### 会话快照
```
定义：每15分钟一次的原始会话记录
内容：该时间段内的原始对话内容，不加工
格式：
  时间段：HH:MM - HH:MM

  用户：xxxxx
  Claude：xxxxx

  用户：xxxxx
  Claude：xxxxx

文件命名：快照-YYYY-M-D-HHMM.md
存放位置：mem/snapshot/
作用：防止意外中断丢失内容
```

### 会话备忘
```
定义：会话结束时基于所有快照整理的结构化摘要
性质：类似会议纪要
文件命名：备忘-YYYY-M-D.md
存放位置：mem/
作用：供下次会话快速恢复上下文，长期保留
```

---

## 二、备忘时间范围

```
以日期为单位，一天一份备忘：
  起始：当天第一条消息
  截止：当天说"归档"时

同一天多次会话：
  → 追加到同一份备忘文件，不新建

跨天会话（如23:00开始，次日01:00结束）：
  → 按开始日期归档
  → 文件名用开始日期

新建备忘的条件：
  当天还没有备忘文件 → 新建

追加备忘的条件：
  当天已有备忘文件 → 追加
```

---

## 三、备忘格式

标题固定不变，无内容时该标题下写"无"。

```markdown
# 会话备忘
日期：YYYY-M-D
时间段：HH:MM - HH:MM
参与方：用户 + 网页Claude / 本地Claude

---

## 一、讨论主题
（本次会话讨论的主题列表）

## 二、关键决策
（每个决策一条）
[决策] 具体内容
[原因] 为什么这样决定

## 三、文件变更
（本次会话新增或修改的文件）
新增：xxx
修改：xxx（改动摘要）

## 四、待办事项
- [ ] 待办1
- [ ] 待办2

## 五、下一步
（下次会话从哪里继续，第一个任务是什么）

## 六、参考快照
（本次会话快照文件列表）
快照-YYYY-M-D-HHMM.md
快照-YYYY-M-D-HHMM.md
```

---

## 四、触发词汇

```
手动快照：  "快照" / "保存快照"
结束归档：  "归档" / "结束会话"
恢复会话：  "继续[项目]" / "恢复上一个session"
```

---

## 五、标准工作流程

```
启动会话
  ↓
执行启动检查流程（见第六章，强制）
  ↓
正常对话
  ↓
每15分钟：Claude主动生成一次会话快照
  ↓
关键时刻可说"快照"手动触发（可选）
  ↓
会话结束说"归档"：
  → 基于所有快照整理生成会话备忘
  → 下载给用户
```

---

## 六、启动检查流程（强制执行，优先于一切任务）

```
每次会话启动时：

第一步：读取 mem/snapshot/ 目录
        找最新快照文件，读取日期

第二步：与今天日期对比
  → 快照日期 < 今天日期（有未归档快照）
        自动整理生成对应日期的备忘文件
        提示用户："已自动生成备忘-YYYY-M-D.md，请确认内容"
  → 无未归档快照
        正常继续

第三步：向用户报告检查结果

第四步：开始执行用户任务
```

---

## 七、中断恢复流程

```
发生中断后：
  说"继续[项目]" → 加载最新快照，从该点继续
  说"开始新session" → 忽略快照，重新开始

恢复优先级：
  有手动快照 → 优先加载最新手动快照
  无手动快照 → 加载最新自动快照
```

---

## 八、文件管理

```
存放位置：
  会话快照：mem/snapshot/
  会话备忘：mem/

目录结构：
  mem/
  ├── 备忘-2026-3-11.md
  ├── 备忘-2026-3-12.md
  └── snapshot/
      ├── 快照-2026-3-11-1400.md
      ├── 快照-2026-3-11-1415.md
      └── 快照-2026-3-11-1430.md

删除规则：
  快照文件和备忘文件均不自动删除
  由用户人工指示Claude删除
```
