# PDF智转器 软件体系架构

## 1. 总体架构

系统采用“脚本层 + 转换层 + 适配层 + 校验层”分层架构。

```text
PDF输入
  -> 检测与提取（detector/extractor/vision）
  -> 两步转换（converter: restore + enhance）
  -> 模型适配（model_adapter）
  -> 质量校验（validator）
  -> Markdown与报告输出（output）
```

## 2. 模块分层

1. 接入层（CLI）
- `src/pdfsc.py`
- 负责参数解析、配置加载、任务调度、批量处理

2. 处理层（Pipeline）
- `src/converter.py`
- 实现 restore/enhance 双阶段转换与降级标记

3. 适配层（Model Adapter）
- `src/model_adapter.py`
- 统一对接 anthropic/openai/qwen/ollama/mock

4. 数据层（Extraction / Metadata）
- `src/extractor.py`
- `src/detector.py`
- `src/metadata.py`
- `src/vision.py`
- `src/image_handler.py`

5. 质量层（Validation）
- `src/validator.py`
- 负责结构验证与幻觉检测

## 3. 运行模式

- `real`：真实模型调用
- `offline`：无外部调用，本地回退
- `mock`：本地 mock 输出，流程联调

状态可观测：
- 转换结果包含 restore/enhance mode
- 批量报告统计模式分布与低质量命中

## 4. 配置与扩展点

扩展点：
- 新模型：在 `model_adapter.py` 增加适配器并注册工厂
- 新验证规则：在 `validator.py` 增加规则检测
- 新运行策略：在 `converter.py` 增加路由与标记策略

## 5. 质量保障机制

- 两步转换防止直接幻觉扩散
- `offline/fallback/mock` 自动 `DRAFT/LOW_QUALITY` 标记
- 测试集覆盖 D2 流程联调与阶段回归
